# Clean Browsing v1.0.0 — First Stable Release

**Release type:** Major
**Extension version:** 1.0.0
**Target:** Firefox (Manifest V2, gecko min 109.0)

## Overview

v1.0.0 is the **first stable release** of Clean Browsing and the first
release off a complete rewrite of the codebase. Everything under the hood
has changed: the old vanilla HTML/CSS/JavaScript build has been replaced
by a modern stack — **Svelte 5** (runes), **Vite 6**, **TypeScript**,
**Tailwind v4**, and **shadcn-svelte** — while keeping the extension
100% local-first with zero network calls, zero telemetry, and no
accounts.

If you're upgrading from v0.5.0 or earlier, your existing data migrates
automatically on first launch.

## What's in the box

### Grid and edit mode

- **24×16 cell grid** of draggable, resizable widgets
- **Edit mode** toggled from the toolbar — jiggle-style animation, dashed
  padding outlines, free-flight drag previews, collision detection on
  every hovered cell
- **Pointer-capture drag and resize** with south / east / southeast
  handles; widgets always land in a valid cell
- **Per-instance layout** persisted to `browser.storage.local` under
  `clean-browsing:layout:v2`; falls back to `localStorage` outside the
  extension context
- **Migration** from the legacy `localStorage` key is one-shot and
  automatic — existing users won't lose their layout

### Built-in widgets

| Widget | What it does |
|---|---|
| 🕒 **Clock** | Locale-aware time display with 12/24h, seconds, AM/PM, flashing colons, DST toggle, and custom padding |
| 📅 **Date** | Day.js formatted date with a wide range of format strings |
| 🔍 **Search** | Multi-engine search bar with Bing, DuckDuckGo, Google, and Yahoo |
| 🧮 **Calculator** | Keyboard-friendly four-function calculator |
| 🖼️ **Picture** | User-supplied image tile backed by the built-in image library |

Each widget is implemented as a folder trio (`<Name>.svelte` +
`<Name>Settings.svelte` + `definition.ts`) and registers itself with a
central registry at module load. Adding a new widget is a single
`import` line — see [`docs/WIDGET_DEVELOPMENT.md`](../docs/WIDGET_DEVELOPMENT.md).

### Theming

- **Light and dark modes** driven by `mode-watcher`
- **Design tokens** defined as CSS variables in `src/app.css` and exposed
  to Tailwind through `@theme inline`
- **Project brand palette** sampled from the logo (orange, green, blue,
  lilac) used consistently across both themes
- Every `--ui-*` variable has a paired value under `.dark` — theme
  switching is instant and requires zero JavaScript in widget code

### Settings

- **Per-widget settings dialogs** — each widget ships its own settings
  form, rendered inside a shadcn-svelte Dialog
- **Global widget appearance defaults** — font size, weight, italic,
  underline, text color and opacity, background color and opacity,
  backdrop blur, border radius, overall opacity, text alignment, vertical
  alignment, and padding apply to all widgets at once
- **Backgrounds** — solid colors, gradients, or user-uploaded images
- **Widget presets** for quickly swapping between appearance profiles
- **Storage panel** showing current usage

### Import / export

- **JSON settings export** — download your entire Clean Browsing
  configuration as a single JSON file
- **Image library ZIP export** via `fflate` — pack every stored image
  into a zip for backup or transfer

### Developer experience

- **`npm run dev`** — live dev loop via `vite build --watch` +
  `web-ext run --target=firefox-desktop`; edits to `.svelte` / `.ts` /
  `.css` files hot-reload the extension in Firefox
- **`npm run build`** / **`npm run check`** — production build and
  svelte-check type diagnostics
- **Single-package project** — no monorepo, no workspaces, no subfolder
  indirection

## Breaking changes

This is a greenfield rewrite, so effectively every internal API changed.
**None of the breaking changes are user-facing** — layouts, widget
settings, and image library contents all migrate automatically from
previous versions. If you're a *contributor*, be aware that:

- The old `extension/` tree is gone from `main`. It's preserved at the
  **`legacy-final`** git tag for reference.
- Core files renamed: `newtab.html` → `index.html`, `settings.js` →
  `src/lib/settings/store.svelte.ts`, `widgets.js` → `src/lib/grid/` +
  `src/lib/widgets/`, `styles.css` → `src/app.css` + component-scoped
  `<style>` blocks.
- Widgets are no longer standalone `<script>` tags — they're Svelte
  components registered through a typed registry. See
  [`docs/WIDGET_DEVELOPMENT.md`](../docs/WIDGET_DEVELOPMENT.md).
- The grid is now **24×16**, not 40×24. Saved layouts from v0.5.0 and
  earlier are migrated to fit.

## Installation

### From a GitHub release artifact

1. Download `clean_browsing-1.0.0.zip` (or the signed `.xpi`) from the
   [v1.0.0 release page](https://github.com/dmeim/clean-browsing/releases/tag/v1.0.0).
2. In Firefox, open `about:debugging#/runtime/this-firefox`.
3. Click **Load Temporary Add-on…** and pick the `manifest.json` inside
   the extracted zip (or the `.xpi` file directly).
4. Open a new tab to start using Clean Browsing.

### From source

```bash
git clone https://github.com/dmeim/clean-browsing.git
cd clean-browsing
npm install
npm run build
```

Then load `dist/manifest.json` via `about:debugging`.

### Previous releases

Release notes for v0.1.0 through v0.5.0 in `release-notes/` describe the
**legacy vanilla-JS extension**. That code is preserved at the
`legacy-final` git tag but is no longer built or maintained.
