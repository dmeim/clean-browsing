# Clean Browsing v1.1.0 — Dense Grid

**Release type:** Minor
**Extension version:** 1.1.0
**Target:** Firefox (Manifest V2, gecko min 109.0)

## Overview

v1.1.0 is the first minor release after the v1.0.0 stable rewrite. It
focuses on **placement flexibility**: every widget can now shrink to a
single grid cell, and a new opt-in **Dense Grid** mode doubles the
underlying grid resolution from 24×16 to 48×32 for fine-grained layout
control. No data migration is required — existing layouts continue to
work unchanged.

## What's new

### 1×1 widgets

Every built-in widget (Clock, Date, Search, Calculator, Picture) now
allows resizing all the way down to a **single grid cell**. Previously
each widget declared a `minSize` floor (e.g. Search was locked to 5×2,
Calculator to 3×5). Those floors have been removed across the board, so
you can build slim search bars, two-cell clocks, or any other
ultra-compact layout you want.

Widgets continue to scale their internal content automatically — there
is no separate "compact mode" to enable.

### Dense Grid mode

A new **Grid** tab in the Settings dialog exposes a Density toggle:

- **Normal — 24×16** (default)
- **Dense — 48×32**

Toggling Dense doubles the number of cells in each dimension while
**preserving the visual layout**: every existing widget's position and
size are doubled at the same time, so nothing on screen moves or resizes.
Toggling back to Normal halves them and re-packs any widgets that would
otherwise collide via the existing free-slot finder.

Use Dense mode when you want sub-cell placement precision or to fit more
widgets into the same physical screen area.

### Search widget submit button

The search widget's submit button used to be a fixed 2×2 rem circle,
which looked oversized when the widget was shrunk to a single row. It
now scales with the input row height (capped at 2rem), and the icon
inside scales too. This was a prerequisite for the 1×1 minimum work.

## Fixes

- Search widget submit button no longer breaks the layout when the
  widget is shorter than 2rem.
- Removed an unused `minSize` indirection from the grid resize / store
  paths so the existing `min: 1` clamp is the only enforcement point.

## Breaking changes

None for users. Layouts, settings, widget instances, and the image
library all carry over unchanged from v1.0.x.

For contributors:

- `WidgetDefinition.minSize` is still a valid field on the type but is
  no longer set by any built-in widget. The grid still honors it if a
  third-party widget declares one.
- `gridStore` exports two new members: `isDense` (getter) and
  `setDenseGrid(enabled)`. `setDenseGrid` mutates instance coordinates
  in place inside the existing `beginEdit/commitEdit/cancelEdit`
  transaction, so it integrates with the Settings dialog's
  Save/Cancel flow without extra wiring.

## Installation

### From a GitHub release artifact

1. Download `clean_browsing-1.1.0.zip` (or the signed `.xpi`) from the
   [v1.1.0 release page](https://github.com/dmeim/clean-browsing/releases/tag/v1.1.0).
2. In Firefox, open `about:debugging#/runtime/this-firefox`.
3. Click **Load Temporary Add-on…** and pick the `manifest.json` inside
   the extracted zip (or the `.xpi` file directly).
4. Open a new tab to start using Clean Browsing.

### From source

```bash
git clone https://github.com/dmeim/clean-browsing.git
cd clean-browsing
git checkout v1.1.0
npm install
npm run build
```

Then load `dist/manifest.json` via `about:debugging`.
