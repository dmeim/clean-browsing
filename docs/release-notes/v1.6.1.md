# Clean Browsing v1.6.1

**Release type:** Patch
**Extension version:** 1.6.1
**Target:** Firefox (Manifest V2, gecko min 109.0)

## What's new

### Watchlist column sizing fix

The Watchlist widget now uses true shared table columns for symbols, prices,
changes, volume, and other text fields. Non-sparkline columns size to the
widest visible value in that column, while the Sparkline column expands to
fill the remaining space no matter where it appears in the column order.

This fixes the bug where sparklines stopped short when placed between other
columns and restores clean header/data alignment for mixed watchlist layouts.

## Breaking changes

None. Existing layouts, settings, and widget instances carry over unchanged
from v1.6.0.

## Documentation

- Updated [`docs/widgets/stock-watchlist.md`](../widgets/stock-watchlist.md) —
  clarified shared column sizing and sparkline expansion behavior.

## Installation

### From a GitHub release artifact

1. Download `clean_browsing-1.6.1.zip` (or the signed `.xpi`) from the
   [v1.6.1 release page](https://github.com/dmeim/clean-browsing/releases/tag/v1.6.1).
2. In Firefox, open `about:debugging#/runtime/this-firefox`.
3. Click **Load Temporary Add-on...** and pick the `manifest.json` inside
   the extracted zip (or the `.xpi` file directly).
4. Open a new tab to start using Clean Browsing.

### From source

```bash
git clone https://github.com/dmeim/clean-browsing.git
cd clean-browsing
git checkout v1.6.1
npm install
npm run build
```

Then load `dist/manifest.json` via `about:debugging`.
