# Clean Browsing v1.6.0 — Crypto Support & Watchlist Improvements

**Release type:** Minor
**Extension version:** 1.6.0
**Target:** Firefox (Manifest V2, gecko min 109.0)

## Overview

v1.6.0 adds **cryptocurrency support** to the Stock and Watchlist widgets.
Instead of separate Crypto / Crypto Watchlist widgets, the existing widgets
now handle stocks, ETFs, and crypto through the same Yahoo Finance backend
— just search for `BTC-USD`, `ETH-USD`, or any Yahoo Finance crypto
ticker. The widgets automatically detect the asset type and adjust their
behavior: no "Market closed" badge for 24/7 crypto, smart decimal
formatting for sub-cent coins, and equity-only stats (P/E, Div yield)
hidden when viewing crypto.

This release also adds a **column titles toggle** to the Watchlist and
rewrites the Sparkline component from canvas to SVG for reliable sizing.

## What's new

### Crypto support in Stock & Watchlist widgets

Both widgets now support crypto via Yahoo Finance tickers (`BTC-USD`,
`ETH-USD`, `DOGE-USD`, `SHIB-USD`, etc.). The widgets auto-detect the
asset type from Yahoo's response and adapt:

- **24/7 refresh** — crypto symbols never pause, even when "Pause when
  market closed" is on. For mixed watchlists (stocks + crypto), the
  refresh loop stays active for the crypto symbols.
- **No "Closed" badge** for crypto. Mixed watchlists show "Equities
  closed" instead of "Closed" so it's clear which assets are affected.
- **Smart price formatting** — sub-cent crypto (SHIB, PEPE, etc.)
  displays up to 8 decimal places so you see meaningful prices instead
  of `$0.00`.
- **Equity-only stats hidden** — P/E ratio and Dividend yield are
  automatically filtered out when viewing a crypto symbol.
- **Updated search hints** — the Symbol tab now shows crypto examples
  (`BTC-USD`, `ETH-USD`) alongside exchange-suffix examples.
- **Updated network notices** — both widgets disclose that they fetch
  data for "stocks, ETFs, and crypto."

### Watchlist column titles

A new **Show column titles** toggle in the Watchlist Display tab renders
a header row above the data with abbreviated column labels (Price, Chg,
Chg %, Vol, Mkt Cap, Range). Off by default for a cleaner look.

### Sparkline rewrite (canvas → SVG)

The shared Sparkline component has been rewritten from HTML Canvas to SVG.
The SVG approach uses a `viewBox` coordinate system with
`preserveAspectRatio="none"`, so the sparkline fills its container
reliably in all flex contexts — fixing a longstanding issue where
sparklines didn't fill their column width when placed between other
columns. `vector-effect="non-scaling-stroke"` keeps the line thickness
uniform regardless of container size.

### Watchlist renamed

The widget picker now shows **Watchlist** instead of "Stock Watchlist"
since it handles both stocks and crypto. The widget ID (`stock-watchlist`)
is unchanged — existing layouts carry over without migration.

## Markets backbone additions

New shared utilities in `src/lib/markets/`:

- **`AssetType`** — `"equity" | "crypto" | "etf" | "fund" | "other"`,
  detected automatically from Yahoo's `instrumentType` field (with
  `exchangeName === "CCC"` fallback).
- **`shouldPauseRefresh()`** / **`shouldPauseRefreshMixed()`** — crypto-aware
  market-hours helpers that widgets call instead of raw
  `isUSEquityMarketOpen()`.
- **`formatPriceSmart()`** — adaptive decimal formatter for sub-cent
  prices.
- **`EQUITY_ONLY_STATS`** — set of stat fields to filter out for
  non-equity assets.

## Breaking changes

None. Existing layouts, settings, and widget instances carry over
unchanged from v1.5.3. Old cached quotes lacking the new `assetType`
field default to `"equity"` and populate on the next fetch.

## Permissions

No new permissions. Crypto data comes from the same Yahoo Finance
endpoints already declared:

- `https://query1.finance.yahoo.com/*`
- `https://query2.finance.yahoo.com/*`

## Installation

### From a GitHub release artifact

1. Download `clean_browsing-1.6.0.zip` (or the signed `.xpi`) from the
   [v1.6.0 release page](https://github.com/dmeim/clean-browsing/releases/tag/v1.6.0).
2. In Firefox, open `about:debugging#/runtime/this-firefox`.
3. Click **Load Temporary Add-on...** and pick the `manifest.json` inside
   the extracted zip (or the `.xpi` file directly).
4. Open a new tab to start using Clean Browsing.

### From source

```bash
git clone https://github.com/dmeim/clean-browsing.git
cd clean-browsing
git checkout v1.6.0
npm install
npm run build
```

Then load `dist/manifest.json` via `about:debugging`.
