# Clean Browsing v1.2.0 — Weather Widget

**Release type:** Minor
**Extension version:** 1.2.0
**Target:** Firefox (Manifest V2, gecko min 109.0)

## Overview

v1.2.0 adds the **Weather** widget — current conditions, a configurable
hourly strip, and a multi-day daily forecast, powered by the free, keyless
[Open-Meteo](https://open-meteo.com/) API. This is the first widget in Clean
Browsing that makes outbound HTTP requests, and it ships under the network
rules set out in [`docs/PRIVACY_POLICY.md`](../PRIVACY_POLICY.md): opt-in,
disclosed, scoped to specific hosts, and zero traffic until you actually
configure a location.

## What's new

### Weather widget

A new built-in widget that displays:

- **Current conditions** — temperature, condition label, feels-like temperature.
- **Stat column** — humidity, wind speed, today's max precipitation
  probability, and today's max UV index.
- **Sunrise / sunset row** — today's sunrise and sunset times.
- **Hourly strip** — a configurable window of upcoming hours (3 / 6 / 12 / 24 / 48).
  The strip **auto-centers** when the visible hours fit within the widget
  width, and falls back to a left-aligned horizontal scroll when they don't.
- **Daily strip** — a configurable 3 / 5 / 7-day daily forecast with min / max
  temperatures and condition icons.
- **Custom title** — override the widget header text, useful when
  "Use my location" gives you a generic label you'd rather replace.
- Padding controls and an optional **Compact** mode that hides the hourly
  and daily strips for very small widget sizes.

The widget always caches the full 7-day / 48-hour window from Open-Meteo in a
single request, so toggling **Forecast days** or **Hours ahead** is a pure
re-render — no extra network request. Open-Meteo's free tier counts HTTP
calls, not response size, so pulling the max costs the same as pulling three
days.

### Location picking

Two ways to set a location:

1. **City search.** Type a city, hit Search, pick a result from the
   Open-Meteo geocoder.
2. **Use my location.** A dedicated button in the settings dialog requests
   `navigator.geolocation` exactly once, on click. Geolocation is never
   accessed automatically on mount or on reload.

### Caching and refresh

Forecasts refresh on a configurable interval (15 / 30 / 60 / 120 min). The
last response is cached locally inside the widget's own settings, so
reloading the new tab page renders the previous forecast immediately and
only re-fetches if the cache is older than the interval. A manual refresh
button is available in the widget header.

The cache carries a schema version; a future widget update that adds new
fields will automatically refetch stale caches on first mount rather than
waiting for the refresh interval.

See [`docs/widgets/weather.md`](../widgets/weather.md) for the full settings
reference.

## Network and permissions

Three new permissions land with this release, all exclusively in service of
the Weather widget:

| Permission                               | Why                                                                           |
| ---------------------------------------- | ----------------------------------------------------------------------------- |
| `geolocation`                            | Used only when you click "Use my location" in the Weather widget settings.    |
| `https://api.open-meteo.com/*`           | Used only after you configure a location, on the refresh interval you choose. |
| `https://geocoding-api.open-meteo.com/*` | Used only when you submit a city search in the Weather widget settings.       |

A freshly installed Clean Browsing with no Weather widget on the grid still
makes **zero outbound requests**. A Weather widget that has been added but
never configured also makes zero requests — the first fetch happens when you
finish picking a location.

[`docs/PRIVACY_POLICY.md`](../PRIVACY_POLICY.md) has been updated to list the
new permissions and the two hosts the Weather widget contacts.

## Breaking changes

None for users. Existing layouts, settings, widget instances, and the image
library all carry over unchanged from v1.1.0. No other widgets, stores, or
UI behavior were touched.

## For contributors

- **Widget folder:** `src/lib/widgets/weather/` — `Weather.svelte`,
  `WeatherSettings.svelte`, `WeatherIcon.svelte`, `definition.ts`,
  `api.ts`, `icons.ts`.
- **Inline icons:** `WeatherIcon.svelte` renders Lucide-shaped SVG paths
  from a local `icons.ts` table rather than importing `lucide-svelte`.
  The installed `lucide-svelte@0.475` is Svelte-4-only and is incompatible
  with this project's Svelte 5 runes mode. Future widgets that need icons
  can copy the same pattern.
- **Cache schema:** `WEATHER_CACHE_SCHEMA` in `definition.ts` is bumped
  whenever the cached payload grows new required fields. Weather.svelte's
  cache-insufficiency `$effect` treats older-schema payloads as stale and
  force-refreshes them on first mount.
- **Network discipline:** this release sets the precedent every future
  network-calling widget (Ping Monitor, Mini-Sites, Embeds) will follow.
  See the "Widgets that make network calls" section in
  [`docs/widgets/README.md`](./../widgets/README.md).

## Installation

### From a GitHub release artifact

1. Download `clean_browsing-1.2.0.zip` (or the signed `.xpi`) from the
   [v1.2.0 release page](https://github.com/dmeim/clean-browsing/releases/tag/v1.2.0).
2. In Firefox, open `about:debugging#/runtime/this-firefox`.
3. Click **Load Temporary Add-on…** and pick the `manifest.json` inside
   the extracted zip (or the `.xpi` file directly).
4. Open a new tab to start using Clean Browsing.

### From source

```bash
git clone https://github.com/dmeim/clean-browsing.git
cd clean-browsing
git checkout v1.2.0
npm install
npm run build
```

Then load `dist/manifest.json` via `about:debugging`.
