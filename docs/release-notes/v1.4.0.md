# Clean Browsing v1.4.0 — Notes Widget

**Release type:** Minor
**Extension version:** 1.4.0
**Target:** Firefox (Manifest V2, gecko min 109.0)

## Overview

v1.4.0 adds the **Notes** widget — a markdown sticky-note for quick capture on
the new tab page with interactive task-list checkboxes.

## What's new

### Notes widget

A markdown-powered note widget with:

- **Rendered markdown** — full GFM support including headings, bold, italic,
  links, code blocks, blockquotes, tables, and horizontal rules.
- **Interactive task lists** — `[ ]` and `[x]` items render as native
  checkboxes. Single-click to toggle them; the change persists immediately.
  Supports both standard GFM syntax (`- [ ]`) and bare Obsidian-style (`[ ]`).
- **Double-click to edit** — opens a modal with a monospace textarea and
  Save / Cancel buttons. Editing is only available in view mode; in grid
  edit mode, double-click is suppressed so drag and resize work as expected.
- **Empty-state hint** — a new Notes widget shows a "Double-click to edit"
  placeholder so the interaction is discoverable.
- **Export** — download the current note as a `.md` file from the settings
  dialog.
- **Configurable typography** — font family (sans / serif / mono), font size
  (12–24 px), and independent vertical / horizontal padding.
- **Character counter** — optional char + word count overlay in the
  bottom-right corner.
- **Max character limit** — optional soft cap with a warning near the
  threshold and truncation on save.
- **Security** — rendered HTML is sanitized by DOMPurify. Links open in a
  new tab with `rel="noopener noreferrer"`.

### New dependencies

| Package     | Version | Why                                                        |
| ----------- | ------- | ---------------------------------------------------------- |
| `marked`    | 18.x    | Markdown-to-HTML renderer with GFM task-list support.      |
| `dompurify` | 3.x     | HTML sanitizer — prevents XSS from user-authored markdown. |

Both are runtime dependencies (bundled into the extension JS). No new
permissions are needed.

## Permissions

No new permissions. Notes uses only `storage`, which is already present for
every widget. It makes no network requests.

## Breaking changes

None. Existing layouts, settings, widget instances, and the image library all
carry over unchanged from v1.3.0.

## For contributors

- **Notes folder:** `src/lib/widgets/notes/` — `Notes.svelte`,
  `NotesSettings.svelte`, `markdown.ts`, `definition.ts`.
- **Markdown pipeline:** `renderMarkdown()` in `markdown.ts` runs a
  pre-process (bare `[ ]` → `- [ ]`), then `marked.parse`, then
  `DOMPurify.sanitize`, then a post-process (re-enable task checkboxes,
  harden links).
- **Checkbox toggle:** `toggleTaskAt(source, index)` rewrites the Nth
  task marker in the raw markdown source — the source is the single
  source of truth.

## Installation

### From a GitHub release artifact

1. Download `clean_browsing-1.4.0.zip` (or the signed `.xpi`) from the
   [v1.4.0 release page](https://github.com/dmeim/clean-browsing/releases/tag/v1.4.0).
2. In Firefox, open `about:debugging#/runtime/this-firefox`.
3. Click **Load Temporary Add-on…** and pick the `manifest.json` inside
   the extracted zip (or the `.xpi` file directly).
4. Open a new tab to start using Clean Browsing.

### From source

```bash
git clone https://github.com/dmeim/clean-browsing.git
cd clean-browsing
git checkout v1.4.0
npm install
npm run build
```

Then load `dist/manifest.json` via `about:debugging`.
