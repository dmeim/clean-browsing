# Clean Browsing v1.5.0 — Ping Monitor Widget & Tabbed Widget Settings

**Release type:** Minor
**Extension version:** 1.5.0
**Target:** Firefox (Manifest V2, gecko min 109.0)

## Overview

v1.5.0 ships the **Ping Monitor** widget — a UniFi-style HTTP health-check
dashboard that pings multiple targets on their own schedules and shows live
response times and uptime stats. Alongside it, every widget's settings dialog
has been rebuilt with a **tabbed layout** (Appearance / General / widget-
specific tabs) that mirrors the main app settings modal.

## What's new

### Ping Monitor widget

A service-availability monitor with per-target scheduling:

- **Multi-target dashboard** — add any number of HTTP targets (URLs, IPs,
  hostnames). Each renders as a compact card with a colored status dot, the
  current delay in ms, a mini response-time sparkline, and the target's label.
- **Per-target check intervals** — every target runs on its own timer. Poll
  your DNS server every 5 s and your home lab every hour from the same widget.
- **Duration text inputs** — interval, timeout, and slow threshold are entered
  as human-friendly strings: `5s`, `2000ms`, `30m`, `1h`, `1.5s`. Mixed units
  and decimals are supported. Invalid input shows a red border, valid input
  normalizes on blur (`90000ms` → `90s`).
- **HTTP method per target** — `GET`, `HEAD`, or `POST`. All use
  `fetch(..., { mode: "no-cors" })` so no host permissions are needed; the
  status is derived from whether the round-trip completed and how long it took.
- **Aggregate uptime** — 1-hour and 24-hour rolling uptime percentages across
  all targets, computed from per-target history capped at 200 samples each.
- **Optional transition notifications** — fires once when a target goes from
  reachable to unreachable, and once on recovery. `slow ↔ reachable`
  transitions are skipped to avoid spam. Opt-in, only active while the new tab
  page is open.
- **Manual "Check all" button** — in the widget header, kicks off a check for
  every configured target without waiting for its timer.
- **Legacy schema migration** — any pre-shipping ping-monitor instances that
  used shared top-level `intervalSec`/`timeoutMs`/`slowThresholdMs` fields are
  transparently normalized to per-target values on first render.

### Tabbed widget settings modals

The per-widget settings dialog has been rebuilt with a tabs-on-the-left
layout that mirrors the main app settings modal. Every widget now has two
tabs by default:

- **Appearance** — padding sliders, the existing per-instance appearance
  editor (preset bar, preview, border / glow / shadow / effects), reset
  button, and a placeholder **Inherit global appearance** toggle for a
  future release that will let individual widgets opt out of the global
  defaults.
- **General** — everything else the widget used to show in its flat form.

Specialized widgets can declare additional tabs. Ping Monitor is the first
to use this: it ships with an extra **Targets** tab for managing the
per-target list.

Every shipped widget — Clock, Date, Search, Calculator, Picture, Weather,
Stopwatch, Timer, Notes, Ping Monitor — has been migrated.

### New shared building blocks

Three reusable pieces any future widget can drop into its own settings:

- [`PaddingControl`](../../src/lib/ui/settings/PaddingControl.svelte) —
  vertical / horizontal padding sliders extracted from the copy-paste that
  was living in every widget's settings form.
- [`DurationInput`](../../src/lib/ui/settings/DurationInput.svelte) — text
  input with parse / format / validate + min-max clamping, backed by
  [`parseDuration`](../../src/lib/utils/duration.ts) and `formatDuration`.
- [`WidgetAppearanceTab`](../../src/lib/ui/settings/WidgetAppearanceTab.svelte)
  — the default Appearance tab content. Widgets whose settings don't define
  `paddingV`/`paddingH` (Picture) render without the padding section.

## Permissions

No new permissions. The `notifications` permission already shipped for the
Timer widget in v1.3.0 and is reused for Ping Monitor's optional transition
notifications. Host permissions stay unchanged — Ping Monitor uses
`mode: "no-cors"` for its fetches and therefore does not need per-host
allowlisting.

## Breaking changes

None for end users. Existing layouts, settings, and widget instances carry
over unchanged from v1.4.1.

**For widget authors:** the `WidgetDefinition` type gains a new optional
`settingsTabs?: WidgetSettingsTab<TSettings>[]` field. Existing widgets can
keep using `settingsComponent` (legacy flat layout) — both modes are
supported. See the updated
[`WIDGET_DEVELOPMENT.md`](../WIDGET_DEVELOPMENT.md) for the tabs pattern.

## For contributors

- **Ping Monitor folder:** [`src/lib/widgets/ping-monitor/`](../../src/lib/widgets/ping-monitor/)
  — `PingMonitor.svelte`, `PingTargetCard.svelte`, `PingIcon.svelte`,
  `ping.ts`, `icons.ts`, `definition.ts`, plus `tabs/GeneralTab.svelte` and
  `tabs/TargetsTab.svelte`.
- **Per-target timer architecture:** the widget runs one `setInterval` per
  target, managed as a plain (non-reactive) `Map<string, IntervalId>` that
  is diffed against the desired state on every settings change. Only timers
  whose `intervalMs` actually changed are torn down and recreated — history
  updates are harmless no-ops. Crucial correctness property: plain `Map` /
  `Set` were chosen deliberately over `SvelteMap` / `SvelteSet`. A reactive
  collection inside the timer `$effect` creates a write-read dependency
  cycle that fires fetches at microtask speed.
- **Tabs infrastructure:** [`src/lib/widgets/types.ts`](../../src/lib/widgets/types.ts)
  introduces `WidgetSettingsTab` and `WidgetSettingsTabProps`. The dialog
  detects `def.settingsTabs` and switches to the grid layout mirrored from
  [`src/lib/ui/SettingsDialog.svelte`](../../src/lib/ui/SettingsDialog.svelte);
  otherwise it falls back to the flat body + collapsible Appearance section.
- **Duration utilities:** [`src/lib/utils/duration.ts`](../../src/lib/utils/duration.ts).
  `parseDuration(input)` accepts `ms` / `s` / `m` / `h` suffixes with
  optional whitespace and decimals; returns `null` on invalid input.
  `formatDuration(ms)` picks the largest unit that divides evenly.

## Installation

### From a GitHub release artifact

1. Download `clean_browsing-1.5.0.zip` (or the signed `.xpi`) from the
   [v1.5.0 release page](https://github.com/dmeim/clean-browsing/releases/tag/v1.5.0).
2. In Firefox, open `about:debugging#/runtime/this-firefox`.
3. Click **Load Temporary Add-on…** and pick the `manifest.json` inside
   the extracted zip (or the `.xpi` file directly).
4. Open a new tab to start using Clean Browsing.

### From source

```bash
git clone https://github.com/dmeim/clean-browsing.git
cd clean-browsing
git checkout v1.5.0
npm install
npm run build
```

Then load `dist/manifest.json` via `about:debugging`.
