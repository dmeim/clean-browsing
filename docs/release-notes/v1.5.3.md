# Clean Browsing v1.5.3 — Stock Watchlist Widget

**Release type:** Patch
**Extension version:** 1.5.3
**Target:** Firefox (Manifest V2, gecko min 109.0)

## Overview

v1.5.3 adds the **Stock Watchlist** widget — a multi-ticker table that lets
you track several symbols at a glance with prices, change percentages, and
inline sparklines. It sits on the same `src/lib/markets/` backbone shipped
in v1.5.2 and reuses the Yahoo Finance provider, format helpers, and
market-hours utility without modification.

This is **phase 2 of the Stocks & Crypto suite**. The CoinGecko provider +
Crypto widget (phase 3) and Crypto Watchlist (phase 4) are still planned;
the design doc lives at
[`docs/widgets/wip-stocks.md`](../widgets/wip-stocks.md).

## What's new

### Stock Watchlist widget

A multi-ticker table designed for "10 things I check every morning" rather
than a deep dive into one symbol. Add symbols via search, then the widget
renders a compact row per ticker:

- **Configurable columns** — choose any combination of: Price, Change,
  Change %, Sparkline, Day range, Volume, Market cap. Default is Price,
  Change %, and Sparkline. Column order in the picker is the render order.
- **Inline sparklines** — a new lightweight canvas-based `Sparkline.svelte`
  component in `markets/chart/` draws a HiDPI polyline per row, much
  lighter than spinning up a full `lightweight-charts` instance. Sparkline
  range is configurable: 1D, 5D, or 1M.
- **Sort modes** — Manual (honors the order from the Symbols tab), A-Z,
  Change % (descending), or Price (descending). Sorting never mutates the
  stored symbol order.
- **Drag-to-reorder** — symbols can be reordered via drag and drop in the
  Symbols settings tab. Order matters when sort is set to Manual.
- **Custom title** — the header defaults to "Watchlist" but can be changed
  to any text in the Display tab (same pattern as the Weather widget's
  custom title).
- **Refresh interval** — 1 min / 2 min / 5 min / 15 min, with an optional
  **Pause when market closed** toggle. Default is 2 minutes.
- **Batch fetch** — a new `fetchChartBatch()` utility in the Yahoo Finance
  provider fetches all symbols in parallel via `Promise.allSettled`. Failed
  symbols keep their previously cached data; the widget only errors if all
  fetches fail and there's no cache.
- **Market-closed badge** and **15m delayed badge** in the header, same
  as the Stock widget.
- **Cached on reload** — quotes and sparkline data persist in widget
  settings so the table renders instantly on page load.
- **Error handling** — inline error bar with cached data visible
  underneath, matching the Stock widget pattern.

### Sparkline component (shared)

`src/lib/markets/chart/Sparkline.svelte` is a new shared component
available to any market widget. It takes `data: number[]` and
`color: ChangeColor`, renders a canvas polyline with device-pixel-ratio
awareness, and sizes itself to its CSS container. The Stock Watchlist
uses it for inline row sparklines; the upcoming Crypto Watchlist will
reuse it unchanged.

### Batch provider function (shared)

`fetchChartBatch(symbols, range, signal?)` in `providers/yfinance.ts`
fetches quote + history for multiple symbols in parallel, returning a
`Record<string, ChartFetchResult>` keyed by uppercase symbol. Failed
fetches are silently dropped from the result so callers can fall back to
cached data. Re-exported from `$lib/markets/index.js` for all widgets
to use.

## Breaking changes

None. Existing layouts, settings, and widget instances carry over
unchanged from v1.5.2.

## Permissions

No new permissions. The Stock Watchlist uses the same Yahoo Finance
endpoints already declared for the Stock widget:

- `https://query1.finance.yahoo.com/*`
- `https://query2.finance.yahoo.com/*`

## For contributors

- **Stock Watchlist folder:**
  [`src/lib/widgets/stock-watchlist/`](../../src/lib/widgets/stock-watchlist/) —
  `StockWatchlist.svelte`, `WatchlistIcon.svelte`, `icons.ts`,
  `definition.ts`, plus `tabs/SymbolsTab.svelte` and
  `tabs/DisplayTab.svelte`.
- **Sparkline component:**
  [`src/lib/markets/chart/Sparkline.svelte`](../../src/lib/markets/chart/Sparkline.svelte) —
  shared canvas sparkline used by the watchlist rows.
- **Refresh-loop architecture:** same idempotent mutable-bookkeeping
  pattern as the Stock widget. The refetch effect gates on a
  `JSON.stringify`-d key of `{ symbols, range }` to avoid re-fetching
  when `updateSettings` writes cached data back.

## Dependencies

No new dependencies. The Sparkline component uses the raw Canvas 2D API
(no library). `lightweight-charts` is still used only by `ChartCanvas`.

## Installation

### From a GitHub release artifact

1. Download `clean_browsing-1.5.3.zip` (or the signed `.xpi`) from the
   [v1.5.3 release page](https://github.com/dmeim/clean-browsing/releases/tag/v1.5.3).
2. In Firefox, open `about:debugging#/runtime/this-firefox`.
3. Click **Load Temporary Add-on...** and pick the `manifest.json` inside
   the extracted zip (or the `.xpi` file directly).
4. Open a new tab to start using Clean Browsing.

### From source

```bash
git clone https://github.com/dmeim/clean-browsing.git
cd clean-browsing
git checkout v1.5.3
npm install
npm run build
```

Then load `dist/manifest.json` via `about:debugging`.
