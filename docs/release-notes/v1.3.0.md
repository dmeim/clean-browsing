# Clean Browsing v1.3.0 — Stopwatch & Timer Widgets

**Release type:** Minor
**Extension version:** 1.3.0
**Target:** Firefox (Manifest V2, gecko min 109.0)

## Overview

v1.3.0 adds two new time-tracking widgets — **Stopwatch** and **Timer** — and
ships a full branding overhaul with proper logo and banner assets. Documentation
gets a refresh too, with a streamlined README and a new developer quick-start
guide.

## What's new

### Stopwatch widget

A lap-capable stopwatch with:

- **Start / Pause / Resume / Reset** controls.
- **Persistent state** — a running or paused stopwatch survives page reloads.
  Time is computed from a stored `Date.now()` anchor on every render frame, not
  a background counter, so there is no drift.
- **Click sound** — optional audible tick on button presses (Web Audio API,
  no network).
- Configurable display settings via the widget settings dialog.

### Timer widget

A countdown timer with:

- **Preset durations** and manual input for custom countdowns.
- **Alarm notification** — fires a browser notification when the timer expires,
  even if every tab is closed, via `browser.alarms` and a non-persistent
  background event page.
- **Audible beep** on expiry (Web Audio API, no network).
- **Ring progress indicator** — visual countdown ring that scales with the
  widget size.
- **Persistent state** — a running timer survives page reloads by storing the
  target epoch and recomputing remaining time on mount.

### Branding assets

- New **logo** and **banner** variants in `public/branding/` — color, mono,
  filled, outline, with transparent and solid backgrounds, in both SVG and PNG.
- Favicon, manifest icons, and README banner updated to use the new assets.
- Old `public/icons/` directory removed.

### Documentation

- **README** streamlined — development setup moved to a dedicated
  `docs/DEVELOPER.md` quick-start guide.
- **Widget Development guide** expanded with scaling and responsive layout
  patterns.

## Permissions

Two new permissions land with this release, both exclusively for the Timer
widget:

| Permission      | Why                                                                   |
| --------------- | --------------------------------------------------------------------- |
| `notifications` | Used to alert you when a timer expires, even if the tab is closed.    |
| `alarms`        | Used to schedule timer expiry in the background via `browser.alarms`. |

These permissions are passive until you actually start a timer. A new tab page
with no Timer widget makes zero use of them.

## Breaking changes

None for users. Existing layouts, settings, widget instances, and the image
library all carry over unchanged from v1.2.0.

## For contributors

- **Stopwatch folder:** `src/lib/widgets/stopwatch/` — `Stopwatch.svelte`,
  `StopwatchSettings.svelte`, `clickSound.ts`, `definition.ts`.
- **Timer folder:** `src/lib/widgets/timer/` — `Timer.svelte`,
  `TimerSettings.svelte`, `alarms.ts`, `beepSound.ts`, `definition.ts`.
- **Shared icon module:** `src/lib/widgets/_shared/Icon.svelte` and `icons.ts`
  provide inline SVG icons since `lucide-svelte` is incompatible with Svelte 5
  runes mode.
- **Shared time utilities:** `src/lib/widgets/_shared/time.ts` — formatting
  helpers shared by both widgets.
- **Background script:** `public/background.js` handles `browser.alarms`
  events and fires notifications for expired timers.

## Installation

### From a GitHub release artifact

1. Download `clean_browsing-1.3.0.zip` (or the signed `.xpi`) from the
   [v1.3.0 release page](https://github.com/dmeim/clean-browsing/releases/tag/v1.3.0).
2. In Firefox, open `about:debugging#/runtime/this-firefox`.
3. Click **Load Temporary Add-on…** and pick the `manifest.json` inside
   the extracted zip (or the `.xpi` file directly).
4. Open a new tab to start using Clean Browsing.

### From source

```bash
git clone https://github.com/dmeim/clean-browsing.git
cd clean-browsing
git checkout v1.3.0
npm install
npm run build
```

Then load `dist/manifest.json` via `about:debugging`.
