# Clean Browsing v1.5.2 — Stock Widget & Markets Backbone

**Release type:** Patch
**Extension version:** 1.5.2
**Target:** Firefox (Manifest V2, gecko min 109.0)

## Overview

v1.5.2 introduces the **Stock** widget — a single-ticker market widget with
a live(ish) price, day's change, configurable stats, and an interactive
area chart, powered by Yahoo Finance's public HTTP endpoints. Under the
hood it sits on a new `src/lib/markets/` backbone that the upcoming Stock
Watchlist, Crypto, and Crypto Watchlist widgets will reuse.

This is **phase 1 of the Stocks & Crypto suite**. At the time, the
remaining three widgets were planned for follow-up Z bumps and tracked in
the now-removed `docs/widgets/wip-stocks.md` design doc.

## What's new

### Stock widget

A single-ticker hero widget. Pick one symbol — anything Yahoo Finance
covers (US, UK, Tokyo, Frankfurt, etc.) — and the widget shows price,
change, chart, and stats:

- **Hero block** — symbol (mono) + company name on top, big price below,
  signed change + percent on a third row colored green / red / muted by
  direction. The `↗` / `↘` / `−` icon next to the change row tracks the
  same direction.
- **Interactive area chart** — line + filled gradient via
  [`lightweight-charts`](https://www.tradingview.com/lightweight-charts/),
  with crosshair-on-hover and a dashed reference line at the previous
  close on the 1D range. Color (green / red / muted) tracks the direction
  of the change.
- **Configurable stats grid** — pick any subset of: Change, Change %,
  Day high, Day low, Day range, Open, Previous close, 52-week high,
  52-week low, Volume, Average volume, Market cap, P/E ratio, Dividend
  yield. Order in the picker is the render order.
- **Chart range switcher** — 1D / 5D / 1M / 6M / YTD / 1Y / 5Y. Each
  range is a separate Yahoo request and the result is cached so a reload
  shows the previous chart immediately.
- **Refresh interval** — 30 s / 1 min / 5 min / 15 min, with an optional
  **Pause when market closed** toggle that suspends the loop outside US
  NYSE / NASDAQ trading hours and surfaces a "Closed" badge in the header.
- **Delayed-data badge** — Yahoo's free tier is typically delayed
  15 minutes for US equities. The widget shows the lag in the header so
  you can always tell whether a price is fresh.
- **Symbol search** — debounced search against Yahoo's `/v1/finance/search`
  endpoint with results showing symbol, name, exchange, and security type.
- **One ticker per widget** — to track several symbols, drop several Stock
  widgets on the grid. The shared backbone makes adding a multi-ticker
  Watchlist widget in a follow-up release a layout change, not a rewrite.

### Auto chart hide / show

The Stock widget reads its own size in normal-mode grid cells and toggles
its chart automatically:

- **Below 2×2 normal cells** (1×1 or any thinner / shorter), the chart
  hides — useful for a "minimal mode" that only shows ticker + name +
  price + change.
- **At 3×3 normal cells or larger**, the chart returns automatically.
- **In the 2×2 hysteresis zone**, the user's manual **Show chart** toggle
  in the Display tab wins.

The widget's `minSize` is `1×1` so users can drag it down to a tile-sized
ticker alongside their clock and weather widgets.

### Fluid text scaling

The widget body uses CSS container queries with `cqi`-based `clamp()` so
every text element — symbol, name, badges, hero price, change row, stat
labels, stat values — scales smoothly with widget width. A 1×1 normal
widget shows tight, readable type; a 6×6 widget grows the hero price
without blowing up the labels.

### `markets/` backbone

A new top-level module at [`src/lib/markets/`](../../src/lib/markets/)
contains everything the upcoming Watchlist and Crypto widgets will reuse:

- **[`types.ts`](../../src/lib/markets/types.ts)** — `Quote`,
  `HistoryPoint`, `ChartRange`, `StatField`, `MarketState`,
  `SymbolSearchResult`.
- **[`format.ts`](../../src/lib/markets/format.ts)** — `Intl`-based
  helpers for currency, compact volume, percent, change formatting plus
  a `changeColor()` → `ChangeColor` classifier and a
  `colorVarForChange()` shortcut to the matching CSS variable.
- **[`marketHours.ts`](../../src/lib/markets/marketHours.ts)** —
  `isUSEquityMarketOpen()` / `usEquityMarketState()` using
  `Intl.DateTimeFormat` with the `America/New_York` timezone (Mon–Fri
  09:30–16:00 ET; federal holidays are ignored — Yahoo serves the last
  close on those days, which is the correct rendering anyway).
- **[`providers/yfinance.ts`](../../src/lib/markets/providers/yfinance.ts)**
  — wraps Yahoo Finance's `/v8/finance/chart/<symbol>` endpoint (returns
  quote + history in a single request) and `/v1/finance/search`. Stats
  Yahoo doesn't reliably populate for every ticker (`marketCap`,
  `peRatio`, `dividendYield`, `avgVolume`) come back as `null` and render
  as `—` in the widget rather than `0` or a fake value.
- **[`chart/ChartCanvas.svelte`](../../src/lib/markets/chart/ChartCanvas.svelte)**
  — thin wrapper around `lightweight-charts` v5 with crosshair, dashed
  reference line, and resize-observer-driven sync. The TradingView
  attribution logo is disabled via `attributionLogo: false` (the Apache
  2.0 license doesn't require visible attribution).

### `WidgetProps.gridW` / `gridH`

`WidgetProps<T>` now exposes `gridW: number` and `gridH: number` — the
widget's live size in **normal-mode** grid cells, divided back from
dense-mode storage by `GridItem` so widget code can use a single set of
thresholds regardless of grid density. Tracks the resize preview, not just
the committed size, so size-aware behavior reacts during the drag itself.

This is the mechanism behind Stock's auto chart toggle and is available
to any future widget that wants to branch on its physical size.

### Permissions

Two new host permissions:

- `https://query1.finance.yahoo.com/*` — Stock widget quote + chart +
  symbol search.
- `https://query2.finance.yahoo.com/*` — reserved fallback. Yahoo rotates
  between `query1` and `query2`; declaring both means a future routing
  change won't break the widget without a permissions reissue.

Both are listed in
[`docs/PRIVACY_POLICY.md`](../PRIVACY_POLICY.md), attributed to the Stock
widget. No other host or extension permission changed.

## Breaking changes

None for end users. Existing layouts, settings, and widget instances
carry over unchanged from v1.5.1.

**For widget authors:** `WidgetProps<T>` gained two required fields
(`gridW`, `gridH`). `GridItem` always passes them, so widgets that don't
read them keep working without a code change — the destructuring pattern
`let { settings, updateSettings }: WidgetProps<X> = $props();` continues
to work exactly as before. Widgets that want to opt into size awareness
just add `gridW` / `gridH` to the destructured list.

## For contributors

- **Stock widget folder:**
  [`src/lib/widgets/stock/`](../../src/lib/widgets/stock/) — `Stock.svelte`,
  `StockIcon.svelte`, `icons.ts`, `definition.ts`, plus
  `tabs/SymbolTab.svelte` and `tabs/DisplayTab.svelte`.
- **Markets backbone:** [`src/lib/markets/`](../../src/lib/markets/) — see
  the breakdown above. Public surface is re-exported from
  [`index.ts`](../../src/lib/markets/index.ts).
- **Refresh-loop architecture:** the same idempotent
  mutable-bookkeeping pattern Ping Monitor uses. Two `$effect`s in
  `Stock.svelte` use plain `let lastFetched` and `let currentTimer`
  (not `$state`) to gate network calls and timer creation. Without
  that gate, every successful `updateSettings({ ...settings, cachedQuote, cachedHistory })`
  re-fires the effect (because the prop reference changes), refetches,
  updates again — an infinite microtask loop.
- **Auto chart toggle:** a third `$effect` reads `gridW` / `gridH` and
  writes `settings.showChart` on threshold crossings, with a
  `desired === settings.showChart` short-circuit that prevents the
  same loop pattern. The 2×2 hysteresis zone deliberately leaves the
  user's manual setting alone.
- **`gridW` / `gridH` plumbing:** `GridItem.svelte` divides
  `dispW` / `dispH` (which already track the resize preview) by
  `gridStore.isDense ? 2 : 1` and passes the result, so widget code can
  reason in normal-mode units regardless of grid density.

## Dependencies

- **Added:** `lightweight-charts@^5.1.0` (Apache 2.0). Pulls in
  `fancy-canvas@^2.1.0` as a transitive dep. Both are used only by the
  Stock widget's chart and the shared `ChartCanvas.svelte` wrapper.

## Installation

### From a GitHub release artifact

1. Download `clean_browsing-1.5.2.zip` (or the signed `.xpi`) from the
   [v1.5.2 release page](https://github.com/dmeim/clean-browsing/releases/tag/v1.5.2).
2. In Firefox, open `about:debugging#/runtime/this-firefox`.
3. Click **Load Temporary Add-on…** and pick the `manifest.json` inside
   the extracted zip (or the `.xpi` file directly).
4. Open a new tab to start using Clean Browsing.

### From source

```bash
git clone https://github.com/dmeim/clean-browsing.git
cd clean-browsing
git checkout v1.5.2
npm install
npm run build
```

Then load `dist/manifest.json` via `about:debugging`.
